# Project overview

This repository prototypes a generic component-based hedging and settlement model.

The aim is to separate commercial names from reusable model parts:

- contract part
- call-off
- customer transaction
- market component
- shape component
- risk component
- settlement view

The prototype should demonstrate how market-oriented call-offs can be transformed into customer-oriented settlement output.

## Safety principle

All examples must be synthetic. Use neutral component names and invented numbers only.
