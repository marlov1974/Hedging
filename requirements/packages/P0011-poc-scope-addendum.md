# P0011 PoC scope addendum

This addendum clarifies P0011.

The package is a proof of concept.

Codex does not need to solve production licensing, long-term source rights, SLA, redistribution rights or production data governance.

Use the simplest technically working source for low-volume manual testing.

Public pages, public files or open APIs are acceptable for the PoC.

Do not commit credentials, API keys, private URLs, cookies, session tokens, real customer data or source data dumps.

Do not add high-frequency polling or background scraping.

The research should focus on technical feasibility for a PoC, not production procurement.
