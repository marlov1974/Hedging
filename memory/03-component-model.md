# Component model

The prototype describes settlement output as a composition of generic components.

Core terms:

- `contract part`: a reusable configured part of an agreement model.
- `call-off`: a market-oriented request or allocation that can be transformed into settlement input.
- `customer transaction`: a settlement-facing transaction row derived from model inputs.
- `market component`: a component driven by a market reference.
- `shape component`: a component that represents volume or profile shape.
- `risk component`: a component that represents risk adjustment.
- `settlement view`: the output perspective used for customer-facing calculation review.
